# Write Plan

Use this skill when tackling multi-file changes or complex system modifications.

## Rules
- You must create or update an `implementation_plan.md` before changing any file.
- Outline all target files, dependencies, database schema migrations, and edge cases.
- Define explicit verification and testing criteria.
- Present the plan to the user and wait for approval.
